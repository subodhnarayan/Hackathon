My Key Takeaways

  • File Paths: There are two main types of paths. Relative paths give directions from the current file's location (e.g., `../images/pic.jpg`), while Absolute paths provide the full, exact address (e.g., `C:/Users/photos/pic.jpg` or a full URL). For websites, we almost always use relative paths for our own files.
  • HTML Boilerplate: This is the standard starting code for every HTML page. It's essential because it tells the browser the HTML version, language, character encoding (`UTF-8`), and how to handle mobile viewports. Without it, pages can render unpredictably.
  • `<div>`: A `div` is a generic container, like a bucket, used to group other HTML elements together. This allows you to apply common styling or structure to the entire group.
  • `class` and `id`: These are attributes used to label HTML elements. A `class` is for grouping multiple elements that share the same style (e.g., styling several paragraphs the same way). An `id` is for identifying one single, specific element on a page.

Detailed Notes & Examples

1. File Paths in HTML

A website is a collection of files (HTML, CSS, images) organized in folders. A file path is the address that tells the browser exactly where to find a specific file.

A) Relative File Paths (For Your Own Files)

A relative path gives directions to a file starting from your current location. This is the best way to link your own project files.

Example: Linking to a file in the same folder (`./`)

<img src="./dog.jpeg" alt="dog image">

Here, `./` means "start in the current directory."

Output:

Example: Linking to a file in a parent folder (`../`)

<img src="../../cat.webp" alt="cat image">

Here, `../` means "go up one level." Using it twice (`../../`) means going up two levels to find the file.

B) Absolute File Paths (For External Files)

An absolute path is the full, complete URL to a resource on the web. You use this only when linking to something that is **not** on your own website.

Example: Linking to an image on another website

<img src=https://hips.hearstapps.com/clv.h-cdn.co/assets/16/18/
                gettyimages-586890581.jpg?crop=0.668xw:1.00xh;0.219xw,0&resize=980:*">

This path starts with `https://` and points to a specific image on a different server.

2. HTML Boilerplate Explained

The boilerplate is the standard template you should start every HTML file with. Each line serves a critical purpose to ensure your page works correctly. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- Your content goes here -->
</body>
</html>
  • `<!DOCTYPE html>`: Tells the browser to use the latest HTML5 standards.
  • `<html lang="en">`: The root element, declaring the page's language as English for SEO and accessibility.
  • `<head>`: Contains all the behind-the-scenes metadata for the browser.
  • `<meta charset="UTF-8">`: Sets the character encoding to UTF-8, the universal standard that prevents issues with special characters and emojis.
  • `<meta name="viewport" ...>`: This is essential for responsive design. It tells mobile browsers to set the page width to the device's screen width and use a 100% zoom level.
  • `<title>`: Sets the text that appears in the browser tab.
  • `<body>`: Contains all the visible content of your webpage.

3. `div`, `class`, and `id`

These are used to group and label your content, mainly for applying CSS styles.

  • A `<div>` is a generic "box" or "division" used to group related elements.
  • A `class` is a reusable label for multiple elements you want to style the same way (like a job title, e.g., "Engineer").
  • An `id` is a unique label for one single element on a page (like a Social Security Number).
