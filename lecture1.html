My Key Takeaways

This lecture covered the fundamental building blocks of the internet. Here are my key takeaways and diagrams from the class.

Core Concepts:

  • An IP (Internet Protocol) address is a temporary address your device gets when it's connected to the internet.
  • Your ISP (Internet Service Provider) is who gives you this IP address.
  • DNS (Domain Name System) acts like a phonebook for the internet, translating domain names into IP addresses.
  • An IP address is dynamic (it changes), while a MAC (Media Access Control) address is a permanent hardware address.
  • Port Numbers are used to identify which specific application on a device a message is for.

Core Concepts:

  • IPv4 addresses are 32-bit numbers, allowing for about 4.3 billion unique devices.
  • IPv6 was created to solve the shortage of IPv4 addresses. It uses 128-bit numbers.
  • A MAC address is a 48-bit hexadecimal number.
  • A Port Number is a 16-bit number, with the highest possible port being 65535.

Important Port Numbers for Developers:

  • 21: FTP (File Transfer Protocol)
  • 22: SSH (Secure Shell)
  • 25: SMTP (for sending email)
  • 53: DNS [cite: 6]
  • 80: HTTP (unencrypted web)
  • 443: HTTPS (secure web)

Official Lecture Notes (From First Principles)

The internet can be understood by solving a series of logical problems.

  • Problem 1: Share data between two computers. Solution: A direct physical link.
  • Problem 2: Connect many computers. Solution: A local network with a central switch.
  • Problem 3: Connect many networks. Solution: An "internetwork" connected by routers.
  • Problem 4: Find a specific computer. Solution: A universal IP Address for every device.
  • Problem 5: Send data reliably. Solution: TCP/IP, a set of rules to break data into small, addressed packets.

The DNS Process

DNS translates human-friendly domain names (www.google.com) into computer-friendly IP addresses (142.250.72.206). The process works like this:

  1. Your computer checks its local cache (browser, OS, router) to see if it already knows the IP.
  2. If not, it asks a Recursive Resolver (usually your ISP's server) to find the answer.
  3. The resolver queries the global DNS hierarchy: first a Root Server, then a TLD Server (for .com), and finally the domain's Authoritative Name Server, which gives the final IP address.

Code Examples from this Lecture

To find your MAC address on Windows, you can use these Command Prompt commands.

To see all network configuration details:

ipconfig /all

To see only the MAC addresses:

getmac /v
